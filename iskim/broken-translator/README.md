# 🌐 Broken Translator (번역기 전화 놀이)

> **쓸모없음 지수: ⭐⭐⭐⭐⭐ (5/5)**

## 무엇을 하는 웹사이트인가요?

이 웹사이트는 여러분의 소중한 문장을 **완벽하게 망가뜨립니다**! 🎉

입력한 문장이 다음과 같은 여정을 거칩니다:
```
한국어 → 핀란드어 → 아랍어 → 일본어 → 다시 한국어
```

결과? 원래 의미와는 전혀 다른 엉뚱한 문장! 마치 전화 놀이처럼요.

## 🎯 사용자 시나리오

### 시나리오 1: "첫 방문자의 호기심"
**김개발 씨의 이야기**

1. 🔍 **발견**: 친구가 보내준 링크를 클릭
2. 👀 **첫인상**: 화려한 보라-핑크 그라데이션 배경에 "Where meanings go to die!" 문구를 보고 웃음
3. 💬 **입력**: 예시 버튼 "예시 1"을 클릭해서 "안녕하세요, 저는 개발자입니다." 자동 입력
4. 🔥 **실행**: "번역 망치기 시작!" 버튼 클릭
5. ⏳ **대기**: 로딩 스피너와 함께 "번역 망치는 중..." 메시지 확인
6. 🎪 **결과 구경**: 
   - 각 언어별 번역이 순차적으로 애니메이션과 함께 나타남
   - 핀란드어 → 아랍어 → 일본어를 거쳐 최종 한국어 결과 확인
   - "인사하다, 나는 코드 작가이다" 같은 엉뚱한 결과에 빵 터짐 😂
7. 📊 **망가짐 확인**: "망가짐 지수 87%" - "엄청 망가졌어요!" 메시지 확인
8. 🔄 **재시도**: 자신의 문장을 입력해보고 싶어짐!

## 🚀 로컬 개발

### 1. 설치
```bash
# Node.js 22 사용 (nvm 설치 시)
nvm use

# 의존성 설치
npm install
```

### 2. 환경 변수 설정
```bash
# .env 파일 생성
cp .env.example .env

# .env 파일 편집 (API 키 추가)
# VITE_GOOGLE_TRANSLATE_API_KEY=your_actual_api_key_here
```

**Google Translate API 키 발급 방법:**
1. [Google Cloud Console](https://console.cloud.google.com/) 접속
2. 새 프로젝트 생성
3. "API 및 서비스" → "라이브러리" → "Cloud Translation API" 활성화
4. "사용자 인증 정보" → "API 키 만들기"
5. 생성된 키를 `.env` 파일에 추가

### 3. 개발 서버 실행
```bash
npm run dev
```

브라우저에서 `http://localhost:5173` 열기

### 4. 빌드
```bash
npm run build  # 프로덕션 빌드
npm run lint   # 코드 검사
```
